# Skeleton

Pulsing loading placeholder that mimics content shape while data is fetching.

## Usage

```tsx
import { Skeleton } from '@ds-foundation/react';

// Single text line
<Skeleton className="h-4 w-[250px]" />

// Card placeholder
<div className="flex items-center space-x-4">
  <Skeleton className="h-12 w-12 rounded-full" />
  <div className="space-y-2">
    <Skeleton className="h-4 w-[200px]" />
    <Skeleton className="h-4 w-[160px]" />
  </div>
</div>
```

### With aria-busy wrapper

```tsx
<div aria-busy={isLoading} aria-label="Loading user profile">
  {isLoading ? (
    <div className="flex items-center space-x-4">
      <Skeleton className="h-12 w-12 rounded-full" />
      <div className="space-y-2">
        <Skeleton className="h-4 w-[200px]" />
        <Skeleton className="h-4 w-[150px]" />
      </div>
    </div>
  ) : (
    <UserProfile user={user} />
  )}
</div>
```

### Replacing real content

```tsx
// Table row skeleton
<div className="space-y-2">
  {Array.from({ length: 5 }).map((_, i) => (
    <Skeleton key={i} className="h-10 w-full" />
  ))}
</div>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | — | Set width, height, and shape (e.g., `rounded-full` for circular) |

## Accessibility

- `Skeleton` renders as a plain `<div>` with no ARIA role of its own.
- Wrap skeleton groups in a container with `aria-busy="true"` while loading and remove it when content is ready.
- Provide `aria-label` on the loading container to describe what is being fetched.
- Optionally use an `aria-live="polite"` region to announce when content has finished loading.
