# IconButton

A small icon-only (or icon+text) action button. All variants share a neutral rest state — colour applies only on hover to keep the interface quiet until the user is ready to act.

## Usage

```tsx
import { IconButton } from '@ds-foundation/react';

// Icon-only — always provide title or aria-label
<IconButton variant="danger" icon={<Trash size={16} />} title="Delete row" />

// Icon + label
<IconButton variant="info" icon={<Eye size={16} />} size="sm">View</IconButton>

// Disabled
<IconButton variant="neutral" icon={<Edit size={16} />} disabled title="Edit" />
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'info' \| 'success' \| 'primary' \| 'warning' \| 'danger' \| 'neutral'` | `'neutral'` | Hover colour intent |
| `size` | `'sm' \| 'md'` | `'md'` | sm = 24px, md = 28px |
| `icon` | `React.ReactNode` | — | Icon element (e.g. a Lucide icon at size 16) |
| `children` | `React.ReactNode` | — | Optional label text beside icon |
| `...rest` | `ButtonHTMLAttributes` | — | `disabled`, `onClick`, `title`, `aria-*`, `data-*`, etc. |

## Accessibility

- Always provide `title` or `aria-label` on icon-only buttons — screen readers announce it
- Use `aria-expanded` for toggle controls (e.g. column pickers)
- Focus ring uses the variant's ring token at 2px offset
- `disabled` attribute is passed through via `...rest`
