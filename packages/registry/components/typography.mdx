# Typography

Design system type scale components ‚Äî one export per text style, each rendering the correct semantic HTML element.

## Usage

```tsx
import {
  Display,
  H1,
  H2,
  H3,
  H4,
  H5,
  BodyLarge,
  Body,
  BodySmall,
  Caption,
} from '@ds-foundation/react';

<Display>Treasury Dashboard</Display>
<H1>Account Overview</H1>
<H2>Recent Transactions</H2>
<H3>Filters</H3>
<H4>Date Range</H4>
<H5>Currency</H5>
<BodyLarge>Your balances updated moments ago.</BodyLarge>
<Body>Select a date range to filter transactions.</Body>
<BodySmall>All times are shown in UTC.</BodySmall>
<Caption>Last synced 2 minutes ago</Caption>
```

## Props

All typography components share the same props:

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | ‚Äî | Additional Tailwind classes for overrides |
| `children` | `React.ReactNode` | ‚Äî | Text content |

Standard HTML attributes for the respective element type are also accepted.

## Accessibility

- Heading components (`H1`‚Äì`H5`) render proper semantic heading elements ‚Äî use them to maintain a logical document outline
- Do not skip heading levels (e.g., jump from `H2` to `H4`)
- `Caption` renders a `<span>` with `text-muted-foreground` ‚Äî verify contrast meets 4.5:1 against its background
- All components accept `className` for colour overrides when the muted default is insufficient
