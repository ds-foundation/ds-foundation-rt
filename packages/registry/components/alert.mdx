# Alert

Status or informational banner with semantic color variants built on CVA.

## Usage

```tsx
import { Alert, AlertTitle, AlertDescription } from '@ds-foundation/react';
import { AlertCircle } from 'lucide-react';

<Alert variant="destructive">
  <AlertCircle className="h-4 w-4" />
  <AlertTitle>Error</AlertTitle>
  <AlertDescription>Your session has expired. Please sign in again.</AlertDescription>
</Alert>

<Alert variant="success">
  <AlertTitle>Payment confirmed</AlertTitle>
  <AlertDescription>Your transaction was processed successfully.</AlertDescription>
</Alert>
```

## Props

### Alert

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `"default" \| "destructive" \| "success" \| "info" \| "warning"` | `"default"` | Semantic color style |
| `...rest` | `HTMLAttributes<HTMLDivElement>` | — | Forwarded to the root `<div>` |

### AlertTitle

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `...rest` | `HTMLAttributes<HTMLHeadingElement>` | — | Forwarded to `<h5>` |

### AlertDescription

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `...rest` | `HTMLAttributes<HTMLParagraphElement>` | — | Forwarded to `<div>` |

## Accessibility

- Root element carries `role="alert"` — content is announced immediately to assistive technology (equivalent to `aria-live="assertive"`)
- Always include a non-color indicator (icon or text prefix) alongside the variant color so the message is not conveyed by color alone
- For non-urgent status messages, prefer a component without `role="alert"` to avoid interrupting screen reader flow
