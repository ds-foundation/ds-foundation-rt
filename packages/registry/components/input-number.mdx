# InputNumber

Numeric stepper input with decrement and increment buttons.

## Usage

```tsx
import { InputNumber } from '@ds-foundation/react';

// Uncontrolled
<InputNumber defaultValue={1} min={0} max={100} />

// Controlled integer
const [qty, setQty] = React.useState(1);

<InputNumber
  value={qty}
  onChange={setQty}
  min={1}
  max={99}
  step={1}
/>

// Decimal with precision
<InputNumber
  value={rate}
  onChange={setRate}
  min={0}
  max={1}
  step={0.01}
  precision={2}
/>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `number \| undefined` | — | Controlled value |
| `defaultValue` | `number \| undefined` | — | Uncontrolled initial value |
| `onChange` | `(value: number \| undefined) => void` | — | Called with clamped numeric value on change |
| `min` | `number` | — | Minimum allowed value; decrement button disables at bound |
| `max` | `number` | — | Maximum allowed value; increment button disables at bound |
| `step` | `number` | `1` | Amount added/subtracted per button press |
| `precision` | `number` | `0` | Decimal places for display and rounding |
| `disabled` | `boolean` | `false` | Disables both buttons and the input |
| `className` | `string` | — | Additional classes on the root container |

All other props are forwarded to the underlying `Input` element.

## Accessibility

- The inner input has `role="spinbutton"` with `aria-valuemin`, `aria-valuemax`, and `aria-valuenow`.
- Decrement button has `aria-label="Decrease"`; increment button has `aria-label="Increase"`.
- Buttons are disabled (and aria-disabled) when the value is at its minimum or maximum bound.
- Value is validated and clamped to `[min, max]` on input blur; non-numeric input reverts to the last valid value.
