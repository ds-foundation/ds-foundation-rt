# Checkbox

Boolean or tri-state toggle input built on Radix UI CheckboxPrimitive.

## Usage

```tsx
import { Checkbox } from '@ds-foundation/react';

{/* Uncontrolled */}
<div className="flex items-center space-x-2">
  <Checkbox id="terms" />
  <label htmlFor="terms">Accept terms and conditions</label>
</div>

{/* Controlled */}
<Checkbox
  id="notifications"
  checked={enabled}
  onCheckedChange={setEnabled}
/>

{/* Indeterminate (partial selection) */}
<Checkbox
  id="select-all"
  checked="indeterminate"
  onCheckedChange={handleSelectAll}
/>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `checked` | `boolean \| "indeterminate"` | — | Controlled checked state |
| `defaultChecked` | `boolean` | — | Uncontrolled initial state |
| `onCheckedChange` | `(checked: boolean \| "indeterminate") => void` | — | Called when state changes |
| `disabled` | `boolean` | `false` | Prevents interaction |
| `required` | `boolean` | `false` | Marks field as required for form validation |
| `name` | `string` | — | Form field name |
| `value` | `string` | — | Form submission value |
| `...rest` | `ComponentPropsWithoutRef<CheckboxPrimitive.Root>` | — | Forwarded to Radix root |

## Accessibility

- `role="checkbox"` with `aria-checked` managed automatically by Radix — reflects `true`, `false`, or `"mixed"` for indeterminate
- Always associate with a visible label via `htmlFor` or by wrapping in a `<label>`
- Space key toggles the checkbox
- `aria-disabled` is applied when `disabled` is true
