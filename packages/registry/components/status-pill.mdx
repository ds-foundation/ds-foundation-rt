# StatusPill

A color-coded status badge for payment instruction lifecycle states. Maps workflow stages to semantic feedback colors (neutral, info, success, error).

## Usage

```tsx
import { StatusPill } from '@ds-foundation/react';

<StatusPill status="submitted" />
<StatusPill status="in_payments" />
<StatusPill status="sent_to_bank" />
<StatusPill status="bank_confirmed" />
<StatusPill status="failed" />
```

## Props

| Prop | Type | Description |
|------|------|-------------|
| `status` | `InstructionStatus` | The workflow state to display |

## States

| Status | Label | Color |
|--------|-------|-------|
| `submitted` | Submitted | Neutral |
| `in_payments` | In Payments | Info |
| `first_approval` | 1st Approval | Info |
| `second_approval` | 2nd Approval | Info |
| `sent_to_bank` | Sent to Bank | Success |
| `bank_confirmed` | Confirmed ✓ | Success |
| `failed` | Failed | Error |
| `rejected` | Rejected | Error |

## Accessibility

Uses `role="status"` so transitions are announced by assistive technology. Text label is always visible — color is used as reinforcement, not the sole indicator.
