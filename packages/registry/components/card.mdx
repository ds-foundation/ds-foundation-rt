# Card

Content container with header, body, and footer sections.

## Usage

```tsx
import {
  Card,
  CardHeader,
  CardTitle,
  CardDescription,
  CardContent,
  CardFooter,
} from '@ds-foundation/react';
import { Button } from '@ds-foundation/react';

<Card>
  <CardHeader>
    <CardTitle>Payment Summary</CardTitle>
    <CardDescription>Review your transaction details below.</CardDescription>
  </CardHeader>
  <CardContent>
    <p>Amount: $1,250.00</p>
    <p>Status: Pending</p>
  </CardContent>
  <CardFooter>
    <Button variant="outline">Cancel</Button>
    <Button>Confirm</Button>
  </CardFooter>
</Card>
```

## Props

All sub-components accept `className` and standard HTML div/heading attributes forwarded to their root element.

| Component | Root element | Notable classes |
|-----------|-------------|-----------------|
| `Card` | `<div>` | `rounded border bg-card shadow-sm` |
| `CardHeader` | `<div>` | `flex flex-col space-y-1.5 p-6` |
| `CardTitle` | `<h3>` | `text-2xl font-semibold text-ripple-blue-90` |
| `CardDescription` | `<p>` | `text-sm text-muted-foreground` |
| `CardContent` | `<div>` | `p-6 pt-0` |
| `CardFooter` | `<div>` | `flex items-center p-6 pt-0` |

## Accessibility

- Card is a layout container — add `role="region"` and `aria-labelledby` pointing to `CardTitle` when used as a standalone landmark section
- `CardTitle` renders as `<h3>` — verify this fits your page heading hierarchy
- When the entire card is clickable, wrap in a `<button>` or `<a>` and ensure the accessible name describes the action
