# FreshnessChip

Indicates data recency: hidden when fresh, amber when data is aging (watch), red when stale. Includes an optional refresh action in the stale state.

## Usage

```tsx
import { FreshnessChip, deriveFreshnessState } from '@ds-foundation/react';

const state = deriveFreshnessState(lastUpdatedAt);

<FreshnessChip
  state={state}
  timestamp={lastUpdatedAt}
  onRefresh={() => refetch()}
/>
```

## Props

| Prop | Type | Description |
|------|------|-------------|
| `state` | `'fresh' \| 'watch' \| 'stale'` | Freshness state to display |
| `timestamp` | `Date` | The last-updated timestamp; formatted as relative time |
| `onRefresh` | `() => void` | Optional callback shown as refresh button in stale state only |

## Utility export

```ts
deriveFreshnessState(lastUpdatedAt: Date): FreshnessState
```

Computes `FreshnessState` from a `Date`. Thresholds: watch at 5 min, stale at 15 min.

## Accessibility

The `aria-live="polite"` region is always present in the DOM (even when `fresh` and visually hidden). This prevents an announcement burst when the component first mounts in a live state.
