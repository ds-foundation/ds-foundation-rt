# SP1 — Foundation Design Spec

**Project:** DS v1 — Unified Design System  
**Sub-project:** SP1 — Foundation (Architecture + Tokens)  
**Date:** 2026-04-09  
**Status:** Approved — ready for implementation plan

---

## Goal

Replace the shadcn/ui bridge alias layer with a clean, token-generated semantic alias system. All components reference `--ds-*` CSS custom properties directly via Tailwind `ds.*` utility classes. The DTCG source remains the single source of truth and Figma sync is unaffected.

---

## Architecture

### What does NOT change

- `packages/tokens/src/` — DTCG source files and folder structure are preserved
- Style Dictionary v4 build pipeline (`sd.config.mjs`) — existing transforms and outputs are preserved
- All existing DTCG token categories: primitives (color, radius, shadow, motion, spacing, typography, grid), semantic (light/dark color), density
- The `--ds-color-*`, `--ds-shadow-*`, `--ds-motion-duration-*`, `--ds-motion-easing-*` CSS var names built by Style Dictionary

### What changes

| File/Package | Change |
|---|---|
| `packages/tokens/src/primitives/zindex.tokens.json` | **CREATE** — new z-index primitive token file |
| `packages/tokens/sd.config.mjs` | **MODIFY** — add new `semantic-alias` CSS output platform |
| `packages/react/src/bridge.css` | **DELETE** |
| `packages/react/src/bridge-dark.css` | **DELETE** |
| `packages/react/tailwind.config.ts` | **REPLACE** — remove bridge-dependent aliases, add `ds.*` color aliases |
| `packages/react/src/styles.css` | **MODIFY** — import new semantic alias CSS instead of bridge files |
| All `packages/react/src/*.tsx` (~70 files) | **MIGRATE** — replace shadcn/ui class references with `ds.*` equivalents |

## Section 1: Token Infrastructure

### 1a. New z-index tokens

Create `packages/tokens/src/primitives/zindex.tokens.json`:

```json
{
  "$schema": "https://www.designtokens.org/schemas/2025.10/format.json",
  "$description": "Z-index scale — 10 layers for stacking context management.",
  "z": {
    "$type": "number",
    "base":     { "$value": 0 },
    "raised":   { "$value": 10 },
    "dropdown": { "$value": 100 },
    "sticky":   { "$value": 200 },
    "overlay":  { "$value": 300 },
    "modal":    { "$value": 400 },
    "popover":  { "$value": 500 },
    "toast":    { "$value": 600 },
    "tooltip":  { "$value": 700 },
    "max":      { "$value": 9999 }
  }
}
```

Style Dictionary with `prefix: 'ds'` and `name/kebab` outputs: `--ds-z-base`, `--ds-z-raised`, `--ds-z-dropdown`, etc.

Add `'src/primitives/zindex.tokens.json'` to the `PRIMITIVES` array in `sd.config.mjs`.

### 1b. New semantic alias CSS file

Create `packages/react/src/semantic.css` — a static, hand-maintained file that replaces `bridge.css`. It exposes short-form semantic aliases that map to the full DTCG-built CSS vars. It is NOT auto-generated by Style Dictionary. Changes to it are intentional and code-reviewed.

A matching `packages/react/src/semantic.dark.css` scoped to `[data-theme="dark"]` replaces `bridge-dark.css`.

## Section 2: Tailwind Config Replacement

`packages/react/tailwind.config.ts` is fully replaced. The new config:

- Removes all bridge-dependent color aliases (`background`, `foreground`, `primary`, `muted`, `accent`, `popover`, `card`, `destructive`, etc.)
- Removes all primitive palette aliases (`gray.*`, `blue.*`, `green.*`, `purple.*`, `pink.*`, `orange.*`)
- Adds `ds.*` color aliases pointing to semantic alias vars
- Keeps `borderRadius`, `keyframes`, `animation`, `content` settings
- Removes `darkMode: ['class']` — dark mode is handled by `[data-theme="dark"]` attribute on `<html>`, not a class

```ts
// packages/react/tailwind.config.ts
import type { Config } from 'tailwindcss';
import animate from 'tailwindcss-animate';

export default {
  // Dark mode activates on [data-theme="dark"] attribute on <html>.
  // This keeps the `dark:` Tailwind variant working without a class toggle.
  darkMode: ['selector', '[data-theme="dark"]'],
  content: {
    relative: true,
    files: ['./src/**/*.{ts,tsx}'],
  },
  theme: {
    extend: {
      fontFamily: {
        sans: ['Inter', 'Arial', 'sans-serif'],
      },
      // ring-offset uses ringOffsetColor, not colors — must be listed separately
      ringOffsetColor: {
        'ds-surface': 'var(--ds-surface)',
        'ds-bg':      'var(--ds-bg)',
      },
      colors: {
        ds: {
          bg:               'var(--ds-bg)',
          surface:          'var(--ds-surface)',
          'surface-up':     'var(--ds-surface-up)',
          overlay:          'var(--ds-overlay)',
          sunken:           'var(--ds-sunken)',
          text:             'var(--ds-text)',
          'text-muted':     'var(--ds-text-muted)',
          'text-sub':       'var(--ds-text-sub)',
          'text-disabled':  'var(--ds-text-disabled)',
          'text-inverse':   'var(--ds-text-inverse)',
          border:           'var(--ds-border)',
          'border-strong':  'var(--ds-border-strong)',
          'border-focus':   'var(--ds-border-focus)',
          primary:          'var(--ds-primary)',
          'primary-hover':  'var(--ds-primary-hover)',
          'primary-fg':     'var(--ds-primary-fg)',
          success:          'var(--ds-success)',
          warning:          'var(--ds-warning)',
          danger:           'var(--ds-danger)',
          info:             'var(--ds-info)',
        },
      },
      boxShadow: {
        'ds-sm':  'var(--ds-shadow-sm)',
        'ds-md':  'var(--ds-shadow-md)',
        'ds-lg':  'var(--ds-shadow-lg)',
        'ds-xl':  'var(--ds-shadow-xl)',
        'ds-2xl': 'var(--ds-shadow-2xl)',
      },
      transitionDuration: {
        'ds-fast':   'var(--ds-motion-fast)',
        'ds-normal': 'var(--ds-motion-normal)',
        'ds-slow':   'var(--ds-motion-slow)',
      },
      transitionTimingFunction: {
        'ds-default': 'var(--ds-ease-default)',
        'ds-in':      'var(--ds-ease-in)',
        'ds-out':     'var(--ds-ease-out)',
        'ds-in-out':  'var(--ds-ease-in-out)',
        'ds-spring':  'var(--ds-ease-spring)',
      },
      borderRadius: {
        lg: 'var(--ds-radius-lg)',
        md: 'var(--ds-radius-md)',
        sm: 'var(--ds-radius-sm)',
      },
      keyframes: {
        'accordion-down': {
          from: { height: '0' },
          to: { height: 'var(--radix-accordion-content-height)' },
        },
        'accordion-up': {
          from: { height: 'var(--radix-accordion-content-height)' },
          to: { height: '0' },
        },
      },
      animation: {
        'accordion-down': 'accordion-down 0.2s ease-out',
        'accordion-up': 'accordion-up 0.2s ease-out',
      },
    },
  },
  plugins: [animate],
} satisfies Config;
```

## Section 3: Component Class Migration

All ~70 component `.tsx` files in `packages/react/src/` are migrated. The mapping is deterministic — every old class has exactly one new equivalent.

### Canonical mapping table

| Old (bridge / primitive) | New (`ds.*` semantic) |
|---|---|
| `bg-background` | `bg-ds-surface` |
| `bg-card` | `bg-ds-surface` |
| `bg-muted` | `bg-ds-bg` |
| `bg-accent` | `bg-ds-surface-up` |
| `bg-popover` | `bg-ds-surface` |
| `bg-primary` | `bg-ds-primary` |
| `bg-destructive` | `bg-ds-danger` |
| `text-foreground` | `text-ds-text` |
| `text-muted-foreground` | `text-ds-text-muted` |
| `text-accent-foreground` | `text-ds-text` |
| `text-popover-foreground` | `text-ds-text` |
| `text-card-foreground` | `text-ds-text` |
| `text-primary-foreground` | `text-ds-text-inverse` |
| `text-destructive` | `text-ds-danger` |
| `text-secondary-foreground` | `text-ds-text-muted` |
| `border-border` | `border-ds-border` |
| `border-input` | `border-ds-border` |
| `ring-ring` | `ring-ds-border-focus` |
| `ring-offset-background` | `ring-offset-ds-surface` |
| `divide-border` | `divide-ds-border` |
| Hardcoded `text-gray-*` / `bg-gray-*` | Nearest semantic equivalent per context |
| Hardcoded `focus-visible:ring-blue-500` | `focus-visible:ring-ds-border-focus` |

### Migration approach

Each component file is migrated individually. After migration, the file must contain zero references to: `bg-background`, `bg-card`, `bg-muted`, `bg-accent`, `bg-popover`, `text-foreground`, `text-muted-foreground`, `border-border`, `ring-ring`, `ring-offset-background`, `text-destructive`, `bg-destructive`, or any `var(--background)` / `var(--foreground)` / `var(--muted*)` / `var(--ring*)` references.

A post-migration grep validates zero bridge var references remain.

## Success Criteria

1. `packages/tokens` builds successfully with z-index tokens included
2. `semantic.css` and `semantic.dark.css` are generated and contain all alias vars listed above
3. `bridge.css` and `bridge-dark.css` no longer exist
4. `grep -r "var(--background)\|var(--foreground)\|var(--muted\|var(--ring\|var(--border\|var(--primary\|var(--destructive" packages/react/src/` returns zero matches
5. `grep -r "bg-background\|bg-card\|text-foreground\|text-muted-foreground\|border-border\|ring-ring" packages/react/src/` returns zero matches
6. TypeScript compiles with zero errors (`tsc --noEmit`)
7. All components render correctly in the docs app (light + dark)
